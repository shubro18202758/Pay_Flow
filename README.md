### What PayFlow Does

PayFlow is a **6-stage real-time fraud detection pipeline** that processes every banking transaction through:

1. **рҹ”Қ Ingestion & Validation** вҖ” Schema-checked with CRC32 integrity, timestamp-normalized, amount-to-paisa converted
2. **рҹ§  ML Feature Scoring** вҖ” 36-dimensional behavioral feature extraction вҶ’ GPU-accelerated XGBoost classification вҶ’ 3-tier dynamic risk routing
3. **рҹ•ёпёҸ Graph Intelligence** вҖ” Transaction graph investigation using mule detection, cycle analysis, centrality scoring, and a 3-layer Graph Attention Network (GAT)
4. **рҹӨ– AI Forensic Agent** вҖ” Qwen 3.5 (9B, locally hosted) runs a multi-step LangGraph investigation with tool-calling, chain-of-thought reasoning, and evidence synthesis
5. **вҡЎ Circuit Breaker** вҖ” Weighted multi-model consensus (ML + GNN + Graph) triggers node freezes at вүҘ80% confidence
6. **рҹ”җ Blockchain Verdict** вҖ” Every decision signed with Ed25519, hash-chained to an immutable ledger with Merkle checkpoints and Zero-Knowledge Proofs

### How It Addresses the Problem

| Problem | PayFlow's Answer |
|---------|-----------------|
| Flat tabular rules miss graph patterns | **MultiDiGraph** with Johnson's cycle detection, mule star-pattern scanning, betweenness centrality anomaly detection |
| Black-box alerts | **Full pipeline transparency UI** вҖ” every ML feature, every graph finding, every AI thinking step shown in real-time |
| No immutable evidence | **4-layer cryptographic ledger** вҖ” SHA-256 hash-chain + Ed25519 signatures + Merkle tree checkpoints + ZKP proofs |
| Batch-mode lag | **Sub-second streaming** вҖ” CDC polling at 10ms, SSE push to dashboard, 50вҖ“100 events/sec on a single laptop |
| No investigator tooling | **AI agent with tools** вҖ” queries graph DB, retrieves audit logs, checks freeze status, analyzes unstructured text |

### рҹҸҶ What Makes PayFlow Different

</td></tr>
<tr><td>

**1. Zero Black Boxes вҖ” Complete Algorithmic Transparency**

Every pipeline stage explains itself in the UI. The **Pipeline Transparency X-Ray** panel shows:
- Which of the 36 ML features contributed most to the risk score
- What graph patterns were detected (mules, cycles, centrality anomalies)
- The multi-model consensus scores (ML vs GNN vs Graph evidence)
- The LLM's full chain-of-thought reasoning, tools called, and evidence cited
- The final verdict with confidence, typology classification, and recommended action

**2. Embedded Blockchain вҖ” No External Infrastructure**

Unlike Hyperledger/Tendermint solutions requiring JVM/Go runtimes and 500MB+ RAM, PayFlow's ledger runs in **< 20 MB** using SQLite + PyNaCl + pymerkle. Every fraud verdict is:
- SHA-256 hash-chained (tamper detection)
- Ed25519 signed (non-repudiation)
- Merkle-checkpointed every 100 blocks (regulatory reporting)
- ZKP-anchored (prove risk вүҘ threshold without revealing actual score)

**3. Local-First AI вҖ” No Cloud API Dependency**

Qwen 3.5 (9B) runs entirely on-premises via Ollama. 4-bit Q4_K_M quantization fits in 5.5 GB VRAM. This means:
- **Zero data exfiltration** вҖ” no customer data leaves the bank
- **No API costs** вҖ” no per-token billing
- **Regulatory compliant** вҖ” satisfies RBI data localization norms

**4. GPU Concurrency Orchestration**

A priority queue manages LLM, GNN, and ML training on a single 8 GB GPU:
- Dynamic KV-cache scaling (16K вҶ’ 8K вҶ’ 4K tokens based on VRAM pressure)
- Automatic CPU fallback for GNN/ML when VRAM is critical
- Hysteresis thresholds prevent thrashing between states

**5. Adversarial Simulation Engine**

Built-in threat simulator generates **4 attack typologies** (UPI mule networks, circular laundering, velocity phishing, SWIFT heist probe) with randomized realistic parameters. A **Random Attack Generator** auto-fills custom events with Indian banking data (18 cities, 10 IFSC prefixes, 12 fraud scenarios).

### Installation

```bash
# 1. Clone the repository
git clone https://github.com/shubro18202758/Pay_Flow.git
cd Pay_Flow

# 2. Create Python virtual environment
python -m venv .venv
.venv\Scripts\Activate.ps1    # Windows
# source .venv/bin/activate   # Linux/Mac

# 3. Install Python dependencies
pip install -e ".[dev]"

# 4. Deploy Ollama + Qwen 3.5 model
ollama create payflow-qwen -f scripts/Modelfile

# 5. Install frontend dependencies
cd frontend/app
npm install
cd ../..

# 6. Launch PayFlow
python main.py --serve --events 5000 --fraud-ratio 0.08

# 7. Start frontend dev server (separate terminal)
cd frontend/app
npx vite --port 3006
```
